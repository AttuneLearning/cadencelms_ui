# Commerce UI System Design

**Version:** 1.0.0  
**Date:** January 17, 2026  
**Status:** Draft Specification

---

## Overview

The Commerce System is a **standalone UI module** that handles all payment and financial transactions related to the LMS. It operates as a **loosely-coupled service** that receives events from the LMS but maintains its own state and business logic.

### Design Principles

1. **Independence** - Commerce UI can function without LMS being available
2. **Event-Driven** - Receives completion events, enrollment requests via message queue
3. **Audit Trail** - Every financial transaction is logged and traceable
4. **Multi-Tenant Ready** - Supports multiple organizations/departments with separate billing

## User Stories

### Epic 1: Learner Payments

#### US-1.1: Course Registration Payment
> **As a** learner  
> **I want to** pay for course registration  
> **So that** I can access the course content

**Acceptance Criteria:**
- View course price before checkout
- Select payment method (card, bank, payment plan)
- Receive confirmation email with receipt
- Automatic enrollment upon successful payment
- Handle failed payments gracefully with retry option

#### US-1.2: Program Bundle Purchase
> **As a** learner  
> **I want to** purchase a program (bundle of courses)  
> **So that** I can save money compared to individual courses

**Acceptance Criteria:**
- View program contents and total savings
- Compare individual vs bundle pricing
- Apply available discounts/coupons
- Payment plan options for expensive programs
- Track progress across all courses in program

#### US-1.3: Department Subscription
> **As a** learner  
> **I want to** subscribe to a department's full course catalog  
> **So that** I can access all current and future courses

**Acceptance Criteria:**
- Monthly/annual subscription options
- View all included courses
- Auto-renewal management
- Cancellation and refund policy
- Grace period for payment failures

#### US-1.4: Payment History
> **As a** learner  
> **I want to** view my payment history  
> **So that** I can track my spending and download receipts

**Acceptance Criteria:**
- List all transactions with status
- Filter by date range, type, status
- Download individual or bulk receipts (PDF)
- View active subscriptions and next billing date

---

### Epic 2: Instructor Payouts

#### US-2.1: Earnings Dashboard
> **As an** instructor  
> **I want to** see my earnings overview  
> **So that** I can track my income from courses

**Acceptance Criteria:**
- Current balance available for withdrawal
- Pending earnings (awaiting completion events)
- Historical earnings by period
- Breakdown by course/program
- Projected earnings based on current enrollments

#### US-2.2: Guidepost Completion Bonuses
> **As an** instructor  
> **I want to** receive bonuses when learners hit guideposts  
> **So that** I'm incentivized to create engaging content

**Acceptance Criteria:**
- View configured guidepost bonuses per course
- Track learner progress toward guideposts
- Automatic bonus calculation upon guidepost completion
- Clear audit trail of bonus triggers

#### US-2.3: Course Completion Payouts
> **As an** instructor  
> **I want to** receive payment when learners complete my course  
> **So that** I'm rewarded for successful outcomes

**Acceptance Criteria:**
- Base payout per completion
- Bonus multipliers for high completion rates
- Quality bonuses (based on ratings/reviews)
- Time-based bonuses (early completions)

#### US-2.4: Payout Request
> **As an** instructor  
> **I want to** withdraw my earnings  
> **So that** I can receive my money

**Acceptance Criteria:**
- Minimum payout threshold
- Multiple payout methods (bank transfer, PayPal, etc.)
- Tax document generation (1099, W-9)
- Payout schedule options (immediate, weekly, monthly)
- Transaction fees displayed clearly

---

### Epic 3: Admin Commerce Management

#### US-3.1: Pricing Configuration
> **As an** admin  
> **I want to** configure course and program pricing  
> **So that** I can manage revenue and promotions

**Acceptance Criteria:**
- Set base prices for courses/programs
- Create discount codes and promotions
- Configure payment plan options
- Set regional pricing
- Schedule price changes

#### US-3.2: Instructor Compensation Setup
> **As an** admin  
> **I want to** configure instructor compensation rules  
> **So that** payouts are calculated automatically

**Acceptance Criteria:**
- Set base payout percentages
- Configure guidepost bonus rules
- Set completion bonus tiers
- Define quality bonus criteria
- Override rules for specific instructors

#### US-3.3: Financial Reports
> **As an** admin  
> **I want to** view financial reports  
> **So that** I can track revenue and expenses

**Acceptance Criteria:**
- Revenue by period, course, department
- Payout obligations and projections
- Refund rates and reasons
- Payment failure analysis
- Tax reporting exports

#### US-3.4: Refund Management
> **As an** admin  
> **I want to** process refund requests  
> **So that** I can handle customer issues fairly

**Acceptance Criteria:**
- View refund requests with reason
- Approve/deny with notes
- Partial refund support
- Automatic enrollment reversal on full refund
- Refund impact on instructor payouts

## Data Models

### Transaction

```typescript
interface Transaction {
  id: string;
  type: 'payment' | 'refund' | 'payout' | 'bonus';
  status: 'pending' | 'processing' | 'completed' | 'failed' | 'cancelled';
  
  // Parties
  userId?: string;          // Learner (for payments)
  instructorId?: string;    // Instructor (for payouts)
  
  // Financial
  amount: number;
  currency: string;
  fee: number;
  netAmount: number;
  
  // References
  courseId?: string;
  programId?: string;
  subscriptionId?: string;
  paymentMethodId?: string;
  
  // Metadata
  description: string;
  metadata: Record<string, any>;
  
  // Timestamps
  createdAt: Date;
  processedAt?: Date;
  completedAt?: Date;
}
```

### InstructorEarning

```typescript
interface InstructorEarning {
  id: string;
  instructorId: string;
  
  type: 'enrollment' | 'completion' | 'guidepost' | 'bonus';
  status: 'pending' | 'available' | 'paid';
  
  // Source
  courseId: string;
  learnerId?: string;      // For completion-based earnings
  guidepostId?: string;    // For guidepost bonuses
  
  // Financial
  amount: number;
  bonusMultiplier: number;
  finalAmount: number;
  
  // Audit
  calculationDetails: {
    baseRate: number;
    bonuses: Array<{ type: string; amount: number; }>;
  };
  
  // Timestamps
  earnedAt: Date;
  availableAt: Date;       // After hold period
  paidAt?: Date;
  payoutId?: string;
}
```

### GuidepostRule

```typescript
interface GuidepostRule {
  id: string;
  name: string;
  
  // Trigger
  triggerType: 'percentage' | 'module' | 'completion' | 'time';
  triggerValue: number;    // e.g., 50 for 50%, or module index
  
  // Reward
  rewardType: 'fixed' | 'percentage';
  rewardValue: number;
  
  // Scope
  applyTo: 'all' | 'department' | 'course';
  scopeIds?: string[];     // Department or course IDs
  
  // Status
  isActive: boolean;
  createdAt: Date;
  updatedAt: Date;
}

## API Endpoints (Commerce System)

### Learner Endpoints
```
POST   /commerce/checkout              Create checkout session
GET    /commerce/checkout/:id          Get checkout status
POST   /commerce/payments              Process payment
GET    /commerce/payments              List user payments
GET    /commerce/payments/:id          Get payment details
POST   /commerce/payments/:id/refund   Request refund

GET    /commerce/subscriptions         List user subscriptions
POST   /commerce/subscriptions         Create subscription
DELETE /commerce/subscriptions/:id     Cancel subscription
```

### Instructor Endpoints
```
GET    /commerce/instructor/earnings           Earnings summary
GET    /commerce/instructor/earnings/history   Earnings history
GET    /commerce/instructor/guideposts/:courseId  Guidepost progress
GET    /commerce/instructor/payouts            Payout history
POST   /commerce/instructor/payouts            Request payout
GET    /commerce/instructor/payouts/:id        Payout details
```

### Admin Endpoints
```
GET    /commerce/admin/transactions       List all transactions
GET    /commerce/admin/reports/revenue    Revenue reports
GET    /commerce/admin/reports/payouts    Payout reports

GET    /commerce/admin/pricing            Get pricing config
PUT    /commerce/admin/pricing            Update pricing

GET    /commerce/admin/compensation       Get compensation rules
PUT    /commerce/admin/compensation       Update compensation rules

GET    /commerce/admin/refunds            List refund requests
PUT    /commerce/admin/refunds/:id        Approve/deny refund
